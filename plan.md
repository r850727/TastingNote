# 品飲筆記 Web App 實作計畫

根據您提供的參考網站（田染莊紅白酒買入分析），我將協助您把目前在 `喝過的酒.md` 中的靜態文字表格，轉換成一個**動態、美觀且具備互動性的單頁式網頁應用程式 (SPA)**。

## 1. 技術架構與資料管理

- **前端技術**：使用 HTML5、Vanilla CSS 與 Vanilla JavaScript。不使用複雜的框架，以確保輕量、快速且易於維護（符合參考網站的架構）。
- **資料儲存**：將 `喝過的酒.md` 中的表格資料萃取出來，轉換成 JSON 格式並儲存在獨立的 `data.js` 檔案中。例如：
  ```javascript
  const sakeList = [
    { 
      name: "よこやま 純米吟醸 SILVER7 生", 
      category: "純米吟釀", 
      method: "-", 
      price: 1870, 
      currency: "JPY", 
      image: "./ExportBlock.../IMG_5668.jpeg",
      saketime_rank: 15, // S (11-20)
      note: "" 
    },
    // ...
  ];
  ```
- **本機與 NAS 部署 (Synology NAS)**：
  - 開發與本地測試時，由於只需要讀取本地圖片與靜態資料，可直接透過瀏覽器開啟 HTML 檔案，或是使用簡單的 Live Server 即可運行。
  - **未來部署至群暉 (Synology) NAS**：這套無後端、全靜態網頁 (HTML/CSS/JS) 的架構非常適合 NAS。您只需要開啟 NAS 上的 **Web Station** 功能，將專案資料夾（包含 HTML, CSS, JS 及圖片）放進 `web` 共用資料夾中，即可輕鬆完成部署，不需額外設定 Node.js 或資料庫。
## 2. 視覺設計與 UI/UX (Aesthetics)

為了達到您期望的「質感」與「現代感」，將導入以下設計：

- **極致深色模式 (Premium Dark Mode)**：使用深灰黑底色（如 `#0d1117`, `#161b22`）搭配高對比度的螢光/亮色系文字與邊框，營造出高級感。
- **動態列表與彈窗設計 (List & Modal Layout)**：
  - 為了保持畫面的整潔與提升載入速度，一開始的列表將**不顯示圖片**，僅以精簡的卡片或條目呈現。
  - 列表上會顯示酒名、種類徽章 (Badges)、價格與評級。
  - 當您點擊感興趣的酒款時，會跳出一個具備質感的**詳細資訊彈窗 (Modal)**，在彈窗中才會載入並顯示完整的高清酒瓶照片以及詳細筆記。
  - **Saketime 評級徽章**（僅清酒）：依照 Saketime 排名自動轉換為評級徽章（S+、S、A、B、C），並套用對應的專屬顏色（例如 S+ 綠色、A 藍色等）。
  - **微動畫 (Micro-animations)**：滑鼠懸浮 (Hover) 時卡片邊框變色或有微微的浮起效果。
- **現代字體**：引入 Google Fonts (如 Inter 或 Noto Sans TC) 取代瀏覽器預設字體。

## 3. 核心功能實作

- **分類切換 (Tabs)**：
  - 在畫面頂部實作大按鈕，可切換「清酒」、「梅酒」、「威士忌」、「琴酒」等類別。
- **數據儀表板 (Stats Summary)**：
  - 根據當前選擇的類別，動態顯示「總數量」、「平均價格」或特定類別的分佈（例如純米大吟釀有幾支）。
- **篩選與排序 (Filters & Sorting)**：
  - 價格排序：高到低、低到高。
  - **Saketime 評級篩選**：點擊 S+, S, A, B, C 等按鈕快速過濾對應的清酒。
  - 種類篩選：例如在清酒標籤下，可以快速過濾出「生酒」或「純米大吟釀」。
- **我的最愛/購物車 (Favorites / Cart)**：
  - 仿造參考網站的「購物欄」功能，實作一個「我的最愛」抽屜，點擊卡片上的「+」號可以將喜愛的酒款加入收藏清單，方便快速檢視。
- **快速新增工具 (Add New Wine Form)**：
  - 由於是無後端的靜態網頁，我們將在網頁內建一個「新增酒款」的隱藏表單。
  - 您只需在介面上填寫酒名、價格、選擇照片路徑與評級，點擊「產生」後，系統會自動幫您寫好 JSON 格式的程式碼並**一鍵複製到剪貼簿**。
  - 接著您只要將它貼上到 `data.js` 檔案中，就能無痛完成新增，免去手動對齊格式或寫程式碼的麻煩。

## 4. 實作步驟 (Execution Plan)

1. **第一階段：環境建立與資料轉移**
   - 建立 `index.html`, `style.css`, `app.js`, `data.js`。
   - 解析目前的 `喝過的酒.md`，將所有酒款資料轉換為 `data.js` 中的陣列。
2. **第二階段：核心 UI 開發**
   - 撰寫 CSS 基礎變數（色彩、字體）。
   - 實作深色主題背景與卡片版型。
   - 實作圖片顯示與錯誤處理（避免圖片破圖）。
3. **第三階段：互動邏輯開發**
   - 實作分類頁籤切換（Sake, 梅酒等）。
   - 實作卡片動態渲染功能。
   - 實作排序與篩選功能。
4. **第四階段：美化與細節打磨**
   - 加入 Hover 特效與過場動畫。
   - 確保 RWD (響應式設計) 在手機或不同螢幕上都能完美呈現。
